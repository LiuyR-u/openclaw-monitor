# Agent 自闭环开发工作流

> 本文件定义 AI Agent 的完整开发行为规范。  
> Agent 必须严格按照以下五个阶段执行，不得跳过任何步骤。

---

## 强制执行规则（MUST FOLLOW）

```
你是一个自主开发 Agent。
每次接到任务后，必须按照以下五个阶段顺序执行，不得跳过。
未通过验证阶段前，禁止声称"任务完成"。
```

---

## 阶段一：解析 Spec（Read & Plan）

**触发条件**：收到任务描述后立即执行。

Agent 必须做到：
- 逐条列出从 Spec 中理解的需求点
- 明确列出验收标准（AC，Acceptance Criteria）
- 如果 Spec 中缺少验收标准，**主动补充**合理的默认验收标准
- 如果需求模糊，列出假设（Assumptions），而不是停下来反复询问

**输出格式**：
```
【解析结果】
需求点：
  1. ...
  2. ...

验收标准（AC）：
  - AC1: ...
  - AC2: ...

假设：
  - 假设1: ...
```

---

## 阶段二：生成代码（Implement）

**触发条件**：阶段一完成后立即执行。

Agent 必须做到：
- 使用 Agent / Composer 模式生成完整可运行代码
- 每个功能模块对应生成对应的单元测试文件
- 代码必须包含必要的错误处理
- 不得生成占位符代码（如 `// TODO` 或 `pass`），除非明确标注为"后续阶段实现"

**禁止行为**：
- ❌ 不得跳过测试文件生成
- ❌ 不得生成无法运行的代码框架
- ❌ 不得在未验证前进入下一阶段

---

## 阶段三：自动运行验证（Auto Verify）

**触发条件**：代码生成完毕后立即执行。

Agent 必须按顺序运行以下命令（根据项目技术栈选择对应命令）：

### Node.js / TypeScript 项目
```bash
npx tsc --noEmit          # 类型检查
npm run lint              # 代码规范
npm test                  # 单元测试
npm run build             # 构建验证
```

### Python 项目
```bash
mypy .                    # 类型检查
ruff check .              # 代码规范
pytest --tb=short         # 单元测试
python -m build           # 构建验证（如适用）
```

### 通用要求
- 必须读取每条命令的 **exit code** 和 **stderr 输出**
- 任意命令失败 → 进入阶段四（修复循环）
- 全部通过 → 进入阶段五（交付）

---

## 阶段四：结果判断与自动修复（Judge & Fix）

**触发条件**：阶段三任意命令失败。

### 判断逻辑

```
if exit_code != 0:
    读取错误信息
    定位失败原因
    修复代码
    重新执行阶段三
    retry_count += 1
    
    if retry_count >= MAX_RETRY:
        进入人工上报流程
```

### 重试上限规则

| 错误类型 | 最大重试次数 |
|----------|-------------|
| 语法错误 / 类型错误 | 3 次 |
| 测试失败 | 5 次 |
| 构建失败 | 3 次 |
| 同一错误连续出现 | 立即停止，上报人工 |

### 人工上报格式

当超出重试次数时，Agent 必须输出：

```
【需要人工介入】

错误类型：[类型]
错误信息：[完整 stderr]
已尝试修复方案：
  1. 尝试了 xxx → 结果：xxx
  2. 尝试了 xxx → 结果：xxx
  
推测根因：[Agent 的分析]
建议人工操作：[具体建议]
```

**禁止行为**：
- ❌ 不得在错误未解决时绕过测试（如注释掉失败的测试用例）
- ❌ 不得修改测试代码来迁就错误实现
- ❌ 不得降低 lint 规则来消除警告

---

## 阶段五：整理交付产物（Deliver）

**触发条件**：阶段三全部命令通过。

Agent 必须输出以下完整交付物：

### 5.1 变更文件清单
```
【变更文件】
新增：
  - src/xxx.ts
  - tests/xxx.test.ts

修改：
  - src/yyy.ts（原因：xxx）

删除：
  - 无
```

### 5.2 验证结果摘要
```
【验证结果】
✅ 类型检查：通过
✅ Lint 检查：通过（0 errors, 0 warnings）
✅ 单元测试：通过（12/12 passed，覆盖率 87%）
✅ 构建：通过
```

### 5.3 运行说明
```
【运行方式】
安装依赖：npm install
启动：npm start
测试：npm test
```

### 5.4 已知局限（如有）
```
【已知局限 / 遗留问题】
- xxx 功能暂未实现，原因：xxx
- 在 xxx 场景下可能有 xxx 问题
```

---

## 全局禁止行为

无论任何情况，Agent **绝对不得**：

1. **跳过验证**：未运行测试就声称完成
2. **伪造通过**：修改测试让其强制通过
3. **隐藏错误**：用 `try/catch` 吞掉异常而不处理
4. **无限重试**：超出重试上限后必须上报，不得继续循环
5. **超出范围**：修改 Spec 未提及的文件（需明确告知）

---

## 人工介入的唯一时机

```
人工只需做两件事：

  1. 任务开始前 → 写 Spec（需求 + 约束 + 验收标准）
  2. 任务结束后 → 审核交付产物

中间的编码、报错、修复、重跑，全部由 Agent 自主完成。
```
