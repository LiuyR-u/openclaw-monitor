# 监控器能力说明

## ✅ 当前监控器可以识别工具来源

### 工具来源分类:

#### 1. ⚙️ 内置工具
Openclaw自带的工具:
- `read` - 读取文件
- `write` - 写入文件
- `edit` - 编辑文件
- `glob` - 文件搜索
- `grep` - 内容搜索
- `bash/exec` - 执行命令
- `webSearch` - 网页搜索
- `webFetch` - 网页抓取
- `browser` - 浏览器操作

#### 2. 📦 Skill工具
第三方技能:
- `weather` - 天气查询
- `feishu-*` - 飞书相关
- `imap-smtp-email` - 邮件发送
- `report-analyzer` - 报告分析
- `clawhub` - 技能管理
- 其他自定义skill

#### 3. 🌐 Web工具
网络相关:
- `webSearch` - 网页搜索
- `webFetch` - 网页抓取
- `browser` - 浏览器
- `http/fetch` - HTTP请求

## 🔌 是否需要API?

### 当前监控器: ❌ 不需要API

**原因**:
- 只读取本地日志文件
- 纯本地处理,无网络请求
- 不上传任何数据
- 完全离线工作

### 未来扩展可能需要API的场景:

#### 1. 远程监控(可选)
如果需要远程查看监控:
- 需要WebSocket或HTTP API
- 用于远程终端查看
- 当前不需要,本地查看即可

#### 2. 数据分析增强(可选)
如果需要更智能的分析:
- 可接入AI API分析日志
- 自动识别异常模式
- 当前不需要,基础统计已足够

#### 3. 通知推送(可选)
如果需要异常通知:
- 可接入推送API(邮件/飞书/微信)
- 实时告警
- 当前不需要,实时查看即可

## 使用建议

### 日常使用:
```bash
# 监控最新会话
node monitor.js --latest

# 只看工具调用
node monitor.js --latest --filter tools

# 查看统计
node stats.js <日志文件>
```

### 分析工具使用:
通过监控输出,你可以清楚看到:
- 哪些任务用了Skill
- 哪些任务用了Web搜索
- 哪些任务用了内置工具
- 工具调用是否成功
- 错误发生在哪里
